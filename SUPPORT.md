# Support

## Where to ask

| Need | Best channel |
|---|---|
| Usage question, workflow choice, or idea | GitHub Discussions (Q&A or Ideas) |
| Reproducible repository, packaging, documentation, or validation defect | GitHub Issue using the relevant template |
| Proposal for a new independent skill | New skill request issue template |
| Security vulnerability or sensitive disclosure | Follow [SECURITY.md](SECURITY.md); do not post details publicly |
| Contribution guidance | [CONTRIBUTING.md](CONTRIBUTING.md) |

If Discussions are not yet enabled, use an Issue only for reproducible project defects or a minimal request for the appropriate channel.

## Before opening a report

- Search existing issues and discussions.
- Read [FAQ.md](FAQ.md) and [TROUBLESHOOTING.md](TROUBLESHOOTING.md).
- Include the selected skill, installed version, agent or CLI version, operating system, and sanitized reproduction steps.
- Never include secrets, private repositories, customer data, authorization headers, or raw production logs.

## Support boundaries

This repository provides skill instructions and maintenance guidance. It does not provide managed hosting, guaranteed response times, private consulting, or authorization to test third-party systems. Skills should be used only with appropriate access to the code, data, and infrastructure being examined.
