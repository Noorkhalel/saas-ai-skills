# Roadmap

This roadmap describes direction, not a delivery promise. Priorities can change with maintainer capacity, community feedback, and security needs.

## Next

- Keep standalone packaging, policy copies, generated catalogs, report schemas, and workflow contracts synchronized.
- Expand deterministic routing, behavioral, composition, and mutation-regression coverage as skills evolve.
- Improve documentation examples and compatibility guidance using verified loader behavior.
- Review reference freshness and reduce high-value duplicated reference material without coupling packages.

## Later

- Add carefully scoped skills only where a clear primary deliverable is not already owned.
- Improve machine-readable discovery and routing integrations for compatible launchers.
- Publish more documented composition examples that preserve standalone skill behavior.
- Continue improving optional model-evaluation observability without collecting sensitive user or repository content.

## Out of scope

- A monolithic runtime that makes all skills mutually dependent.
- Hidden prompt behavior, executable evaluation fixtures, or mandatory external model access.
- Claims of universal framework, provider, or production compatibility without evidence.

For proposals, start a Discussion when available or use the new-skill and improvement issue templates.
