# Worked examples

1. **TypeScript cycle:** `billing -> notifications -> users -> billing`; list closing import, affected flow, and extract a lower-level contract or reassign ownership rather than a random shared folder.
2. **False unused plugin:** a package absent from imports is loaded by framework configuration; classify Indirectly used and retain after config/build verification.
3. **Monorepo duplicate:** React/util versions differ across workspaces; capture paths/peer constraints and bundle impact before a compatible workspace-policy consolidation.
4. **Transitive vulnerability:** scanner gives exact advisory, version, and `app -> A -> B` path; classify Scanner warning/Confirmed only after advisory and runtime/reachability evidence, then test compatible remediation.
5. **SDK boundary:** domain imports cloud/database SDK; identify adapter/port only if vendor types/errors block policy tests or replacement.
6. **Frontend weight:** a large package supplies one utility; compare platform/smaller maintained alternative, bundle delta, browser support, and migration tests before replacing.
7. **Keep dependency:** proven stable package has security support and tricky edge cases; retaining it is lower risk than a home-grown replacement.
8. **Framework major:** baseline, migration guide, peer/runtime compatibility, isolated branch/group, regression/canary, and rollback plan.
9. **Dependency confusion:** private namespace resolves against public registry; verify registry config and enforce scoped registry/provenance/allowlist.
10. **Critical external API:** payment provider has no deadline/failure isolation; trace blast radius and add timeout/idempotency/explicit fallback-or-fail-closed/runbook.
11. **MCP permissions:** tool can read workspace, execute code, and access production DB unnecessarily; map permissions/data flow and reduce scope/isolate/audit.
12. **PR growth:** new package plus lock change adds large transitive/bundle footprint; report exact diff/path/output and request justification or lighter safe alternative.
