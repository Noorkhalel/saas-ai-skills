# FAQ

## Is this a single skill?

No. This repository is a collection of independent skills.

## Can I install just one skill?

Yes. Use the loader's named skill selection syntax, such as `--skill refactoring-code`.

## Can I install multiple skills?

Yes, if your `skills` CLI version supports multiple selectors in one command. If not, install each skill separately.

## Will adding docs change skill behavior?

No. Root documentation and per-skill READMEs do not change the content of `SKILL.md`.

## Why are there evaluation folders?

They are internal fixtures for quality checks, examples, and benchmark-style validation.

## What should I read first?

Start with `SKILLS.md`, then open the relevant skill folder's `SKILL.md`.
